Security service

Необходимо создать новый сервис, который интегрируется с SAP HR.
Задача сервиса привязать сотрудника к модулю и указать параметры.
Дополнительно к этому сервис реализует выбор метода аутентификации.

  • LDAP (указать имя в домене)
  • Email (установки и смена пароля при помощи отправленной ссылки)
  • Login (указывается логин и пароль, аутентификация средствами самого сервиса)
  • Sms (указывается номер телефона, пользователь получает одноразовые пароли по смс)

    Application portal

    Необходимо реализовать сервис публикации приложений.
    После аутентификации пользователю выводятся иконки с доступными модулями.
    Нажимая на иконку у пользователя запускается нужное приложение.
    Это возможно реализовать следующими способами:

  • Проксирование приложения
    Большинство приложений делятся на клиентскую и серверную часть.
    Если есть возможность, то клиент запускается прямо с портала и работает с сервером портала,
    который переадресовывает запросы на другие сервера.
    Технически не каждый модуль можно так опубликовать.

  • Редирект с ключом доступа
    По служебному каналу дочерний модуль формирует ключ и адрес ссылки.
    Ссылка является одноразовой авторизацией в модуле связанного пользователя.
    Если вариант модификации приложения (добавление ключа) невозможен,
    то портал обращается к своему агенту на сервере модуля и формирует сессию самостоятельно.
